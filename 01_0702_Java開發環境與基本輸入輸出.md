### 概念 4：Java 程式檔與 `class`

#### 概念說明

Java 程式通常寫在副檔名為 `.java` 的檔案中。若檔案內有 `public class HelloJava`，檔名就必須是：

```text
HelloJava.java
```

也就是：

```text
檔名 = public class 名稱 + .java
```

`class` 可以先理解成「程式碼的容器」。Java 需要把程式碼放在 class 裡面，程式才能被編譯與執行。

本課程的範例都會明確標出檔名。例如：


```java
public class HelloJava {
    public static void main(String[] args) {
        System.out.println("Hello Java");

}
```

不用急著理解 `public static void main(String[] args)` 每個字的完整意義。本單元先記住：

### 概念 5：`main` 方法是程式進入點

#### 概念說明

Java 程式可以有很多程式碼，但執行時需要一個開始位置。`main` 方法就是 Java 應用程式的進入點。

這一行可以先視為固定格式：

```java
public static void main(String[] args)
```

需要掌握的是：

- `main` 裡面的程式會依照由上到下的順序執行。
- 每一行可執行敘述通常要用分號 `;` 結尾。
- 程式區塊用 `{ }` 包住。

#### 範例程式

檔名：`ExecutionOrder.java`

```java
public class ExecutionOrder {
    public static void main(String[] args) {
        System.out.println("Step 1");
        System.out.println("Step 2");
        System.out.println("Step 3");
    }
}
```

#### 執行結果

```text
Step 1
Step 2
Step 3
```

### 概念 7：註解 `comment`

### 概念 8：變數 `variable`

#### 概念說明

變數可以想成「有名字的資料盒」。程式需要暫存資料時，就會使用變數。

例如：

- 姓名可以存在 `name`
- 年齡可以存在 `age`
- 身高可以存在 `height`
- 體重可以存在 `weight`

Java 宣告變數時，要先指定資料型態：

```java
資料型態 變數名稱 = 初始值;
```

#### 範例程式

檔名：`VariableDemo.java`

```java
public class VariableDemo {
    public static void main(String[] args) {
        String name = "Amy";
        int age = 20;
        double height = 1.65;

        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Height: " + height);
    }
}
```

#### 命名規則

變數命名建議：

- 使用英文與數字，不要用空白。
- 第一個字通常小寫。
- 多個英文單字使用 camelCase，例如 `userName`、`finalScore`。
- 名稱要有意義，不要只叫 `a`、`b`、`x`。

### 概念 9：資料型態 `data type`

#### 概念說明

資料型態決定變數可以存什麼資料，以及可以做什麼運算。Java 是 strongly typed language，也就是變數宣告時要明確指定 type。

本單元先使用四種常見型態：

| 型態 | 用途 | 範例 |
|---|---|---|
| `String` | 文字 | `"Amy"` |
| `int` | 整數 | `20` |
| `double` | 小數 | `1.75` |
| `boolean` | true / false | `true` |

#### 範例程式

檔名：`DataTypeDemo.java`

```java
public class DataTypeDemo {
    public static void main(String[] args) {

```

#### 型態選擇

| 資料 | 建議型態 | 原因 |
|---|---|---|
| 姓名 | `String` | 姓名是文字 |
| 年齡 | `int` | 年齡通常用整數 |
| 身高 | `double` | 身高可能有小數 |

### 概念 10：基本運算

#### 概念說明

Java 可以進行一般數學運算：

| 運算子 | 意義 | 範例 |
|---|---|---|
| `+` | 加法 | `a + b` |
| `-` | 減法 | `a - b` |
| `*` | 乘法 | `a * b` |
| `/` | 除法 | `a / b` |
| `%` | 取餘數 | `a % b` |

運算時要注意整數與小數。若兩個 `int` 相除，結果可能只留下整數部分。需要小數結果時，建議使用 `double`。

#### 範例程式：成績平均

檔名：`ScoreAverage.java`

```java
public class ScoreAverage {
    public static void main(String[] args) {
        int score1 = 80;
        int score2 = 90;
        int score3 = 85;

        double average = (score1 + score2 + score3) / 3.0;

        System.out.println("Average: " + average);
    }
}
```

#### 範例程式：BMI 計算

BMI 公式：

```text
BMI = 體重 / (身高 * 身高)
```

檔名：`BmiFixed.java`

```java
public class BmiFixed {
    public static void main(String[] args) {
        double height = 1.75;
        double weight = 68.0;
        double bmi = weight / (height * height);

        System.out.println("BMI: " + bmi);
    }
}
```

### 概念 11：鍵盤輸入 `Scanner`

#### 概念說明

如果資料不是寫死在程式裡，而是由使用者輸入，就需要使用輸入工具。Java 初學常用 `Scanner` 讀取鍵盤輸入。

使用 `Scanner` 需要三個步驟：

1. 匯入 `Scanner`
2. 建立 `Scanner` 物件
3. 使用對應方法讀取資料

#### 基本格式

檔名：`InputName.java`

```java
import java.util.Scanner;

public class InputName {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("請輸入姓名：");
        String name = sc.nextLine();

        System.out.println("Hello, " + name);

        sc.close();
    }
}
```

#### 常用讀取方法

| 方法 | 讀取資料 | 範例 |
|---|---|---|
| `nextLine()` | 一整行文字 | `String name = sc.nextLine();` |
| `next()` | 一個單字 | `String word = sc.next();` |
| `nextInt()` | 整數 | `int age = sc.nextInt();` |
| `nextDouble()` | 小數 | `double height = sc.nextDouble();` |

#### 範例程式：輸入姓名與年齡

檔名：`InputProfile.java`

```java
import java.util.Scanner;

public class InputProfile {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("請輸入姓名：");
        String name = sc.nextLine();

        System.out.print("請輸入年齡：");
        int age = sc.nextInt();

        System.out.println("Name: " + name);
        System.out.println("Age: " + age);

        sc.close();
    }
}
```

#### 學習提示

本單元先避免設計「先讀數字，再讀一整行文字」的複雜輸入順序，因為 `nextInt()` 後接 `nextLine()` 容易遇到換行符號問題。整合題會安排成先讀姓名，再讀數字，降低輸入錯誤。

## 常見錯誤與診斷

| 常見錯誤 | 可能原因 | 診斷方式 | 修正方式 |
|---|---|---|---|
| `class HelloJava is public, should be declared in a file named HelloJava.java` | 檔名與 `public class` 不一致 | 看錯誤訊息中的 class 名稱 | 讓檔名與 `public class` 名稱一致 |
| `';' expected` | 忘記分號 | 錯誤通常出現在該行或下一行 | 在 statement 結尾補上 `;` |
| `cannot find symbol Scanner` | 沒有 import | `Scanner` 下方紅線或編譯錯誤 | 檔案最上方加入 `import java.util.Scanner;` |
| BMI 結果不對 | 公式或型態錯誤 | 用手算比對結果 | 使用 `double`，公式寫成 `weight / (height * height)` |
| 程式沒有輸入提示 | 只寫 `nextInt()` 或 `nextDouble()` | 使用者不知道要輸入什麼 | 每次讀取前先用 `System.out.print()` 提示 |
| 輸出擠在同一行 | 使用 `print` 而不是 `println` | 執行結果沒有換行 | 需要換行時使用 `System.out.println()` |
| GitHub 看不到新檔案 | 只有存在本機，還沒 push | 到 GitHub 頁面重新整理後仍看不到 | 執行 commit 與 push，或在 VS Code / GitHub Desktop 按 Push |
| iClass 只貼 repository 首頁 | 沒有貼到當天資料夾 | 連結打開後看不到 `0702` 資料夾內容 | 貼上 `.../tree/main/0702` 形式的連結 |

### 課堂實作題評分規準

| 評分項目 | 4 分 | 3 分 | 2 分 | 1 分 |
|---|---|---|---|---|
| GitHub 設定 | repo 名稱正確，README 已建立，能 push | repo 正確但 push 需協助 | repo 或 README 設定不完整 | 未完成 GitHub 設定 |

| 變數與型態 | 型態選擇正確且命名清楚 | 型態大致正確 | 型態混用或命名不清 | 幾乎未正確使用變數 |
