# 程式設計與資料結構：課程說明與規定

## 課程名稱

程式設計與資料結構

## 課程目標

本課程以 Java 為主要實作語言，從程式設計基礎開始，逐步進入資料結構。課程重點不是只聽懂語法，而是能夠照著講義完成實作、修改程式、除錯，並將程式碼整理到 GitHub。

完成課程後，應能：

1. 使用 VS Code 撰寫、執行與除錯 Java 程式。
2. 使用 GitHub repository 管理課堂作業與程式碼版本。
3. 使用變數、條件判斷、迴圈、方法、陣列與物件導向概念解決基礎問題。
4. 實作基本資料結構，例如 list、stack、queue。
5. 說明程式執行結果，並能根據錯誤訊息修正程式。

## 開發環境

上課主要使用：

- Visual Studio Code
- JDK
- VS Code 外掛：`Extension Pack for Java`
- GitHub repository：`TKU_114-3_DS`

GitHub repository 建立時請勾選：

```text
Add a README file
```

你可以選擇以下任一方式管理 GitHub：

1. VS Code 內建 Source Control，必要時加裝 GitHub 外掛。
2. Git 指令。
3. GitHub Desktop。

## 作業規定

1. 課堂作業期限為 2 天。
2. 課堂作業不接受補交。
3. 作業以 iClass 當天作業區繳交的 GitHub 資料夾連結，以及 GitHub 上的檔案與 commit / push 紀錄為主要認定依據。
4. 程式必須可以執行，不能只上傳截圖。
5. 檔名必須與 `public class` 名稱一致。
6. 作業請放在指定日期資料夾，例如：

```text
TKU_114-3_DS/
└── 0702/
    ├── HealthCalculator.java
    └── ScoreReport.java
```

建議每次完成作業後都做一次 commit，commit message 要能看出本次完成的內容，例如：

```text
Add 0702 health calculator
```

完成後，請到 iClass 當天作業區繳交「當天資料夾」的 GitHub 連結。不要只貼 repository 首頁。

例如 7/12 的作業應繳交：

```text
https://github.com/你的帳號/TKU_114-3_DS/tree/main/0712
```

## 上課方式

上課請照講義操作。每次課程通常會包含：

1. 概念說明。
2. 範例程式。
3. 課堂實作題。
4. 作業或延伸練習。
5. GitHub commit / push。

請務必自己操作一遍。只看別人寫或只複製程式碼，通常無法在考試與後續作業中完成除錯。

## iClass 與 GitHub 繳交規定

每次作業完成後，請確認 GitHub repository 上看得到當天資料夾，並到 iClass 當天作業區繳交該資料夾的 GitHub 連結。

基本流程：

```bash
git status
git add .
git commit -m "作業說明"
git push
```

如果使用 VS Code 或 GitHub Desktop，也要確認最後有完成 `push`。只有存在自己電腦裡、不在 GitHub 上的檔案，原則上不算完成繳交。

iClass 繳交格式：

```text
https://github.com/你的帳號/TKU_114-3_DS/tree/main/日期資料夾
```

範例：

```text
https://github.com/pei/TKU_114-3_DS/tree/main/0712
```

繳交前請檢查：

1. 連結打開後看得到當天資料夾。
2. 當天資料夾中看得到 `.java` 檔案。
3. 不是只貼 `https://github.com/你的帳號/TKU_114-3_DS` 這種 repository 首頁。

## AI 工具使用規定

可以使用 AI 工具協助理解概念、查錯誤訊息、解釋語法或比較不同寫法，但不能直接把 AI 產生的完整作業當成自己的成果繳交。

可以做：

- 請 AI 解釋錯誤訊息。
- 請 AI 說明某一行程式的意思。
- 請 AI 提供類似但不同的練習題。
- 請 AI 協助檢查程式風格。

不可以做：

- 直接貼上題目，要求 AI 產生完整答案後直接繳交。
- 繳交自己無法解釋的程式碼。
- 和他人繳交高度相同的程式碼，卻無法說明差異。

若使用 AI 協助完成作業，請保留自己的修改紀錄，並確保自己能解釋程式如何運作。

## 學術誠信與程式碼相似度

可以討論觀念、錯誤訊息與解題方向，但作業程式碼必須自己完成。

以下情況可能被視為不當繳交：

1. 與他人程式碼高度相同。
2. 只改變變數名稱，但程式結構幾乎完全相同。
3. 直接複製網路或 AI 產生的答案。
4. 無法說明自己繳交的程式碼。

若需要參考外部資料，請理解後改寫成自己的做法。

## 補充建議

建議養成以下習慣：

1. 每完成一小段功能就執行程式，不要全部寫完才測試。
2. 每次錯誤先看錯誤訊息，不要只看紅線。
3. 每個 `.java` 檔只放一個主要 `public class`。
4. 檔名、class 名稱、資料夾名稱都要清楚。
5. 每次上傳 GitHub 後，到 GitHub 網站重新整理確認檔案真的存在。
